B.18 Usage Notes

Loading Parameters in Python 

import yaml
import json

# YAML format
with open("test_results/SM_120/sm120_params.yaml") as f:
    params = yaml.safe_load(f)

# JSON format
with open("test_results/SM_120/sm120_params.json") as f:
    params = json.load(f)

# Example: compute roofline bound
def roofline_tflops(arithmetic_intensity, params):
    peak_tflops = params["peak_tflops"]["fp16_tensor"]
    mem_bw_tbps = params["dram"]["read_bw_measured_gbps"] / 1000
    return min(peak_tflops, arithmetic_intensity * mem_bw_tbps)

Integration with Cycle-Level Simulators

Parameters marked with latency and throughput fields can be directly used as pipeline stage delays in trace-driven simulators. The pipe field identifies the functional unit for dispatch modeling. The issue_rate field (where present instead of throughput) represents the minimum cycles between consecutive issue of the same instruction type to the same pipe, accounting for resource conflicts.

Version Control

Parameter files should be regenerated whenever:

  1. New measurement sections are added
  2. Anomalous measurements are resolved
  3. Toolchain updates enable previously blocked tests
  4. Clock frequency or driver changes affect absolute timings
