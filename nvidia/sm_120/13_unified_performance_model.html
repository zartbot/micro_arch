CHAPTER 13 · SM_120 · GB202 BLACKWELL

Unified Performance Model

This chapter integrates the scattered latency, throughput, and bandwidth measurements from the preceding chapters into a single computable kernel performance predictor for operator simulation. The model follows a roofline-style multi-resource framework parameterized entirely by measured microarchitectural constants from the SM_120 (GB202, RTX PRO 5000 72GB). Unlike a traditional roofline which considers only arithmetic intensity versus memory bandwidth, this model accounts for six independent hardware resource pipes, synchronization overhead, launch costs, and tail effects. Prediction targets are <20% error for compute-bound kernels and <30% for memory-bound kernels, validated against cuBLAS and CUTLASS measured throughput.

13.1 Model Structure

The top-level kernel execution time is governed by the maximum of three independent resource categories, plus fixed overhead terms:

$$T_{\text{kernel}} = \max(T_{\text{compute}},\; T_{\text{memory}},\; T_{\text{sync}}) + T_{\text{launch}} + T_{\text{tail}}$$

Each term decomposes as follows:

13.1.1 Compute Time

$$T_{\text{compute}} = \frac{\text{FLOPS}}{\text{peak}_{\text{TFLOPS}} \times \eta_{\text{compute}}}$$
$$\eta_{\text{compute}} = f(\text{occupancy},\; \text{ILP},\; \text{pipe\_utilization},\; \text{co\_issue\_ratio})$$

The compute efficiency factor $\eta_{\text{compute}}$ captures the deviation from theoretical peak due to insufficient instruction-level parallelism, low occupancy, pipe contention, and issue slot sharing. For a well-tuned kernel (ILP $\geq$ 4, $\geq$ 16 warps/SM), $\eta_{\text{compute}}$ approaches 0.85–1.0 for TC-bound workloads [S9/case_2, S9/case_23].

13.1.2 Memory Time

$$T_{\text{memory}} = \max(T_{\text{global}},\; T_{\text{smem}},\; T_{\text{reg}})$$
$$T_{\text{global}} = \frac{\text{bytes\_transferred}}{\text{BW}_{\text{eff}}(\text{pattern},\; \text{coalescing},\; \text{hit\_rate})}$$
$$T_{\text{smem}} = \frac{N_{\text{smem\_txn}} \times (34 + 2 \times \bar{C}_{\text{bank}})}{W_{\text{throughput}}}$$
$$T_{\text{reg}} = \frac{N_{\text{ldmatrix}} \times 49}{\text{ILP}}$$

The global memory term uses an effective bandwidth $\text{BW}_{\text{eff}}$ that accounts for coalescing degradation (Table 2 below). The shared memory term includes the 34-cycle base latency plus a 2-cycle penalty per conflicting bank way $\bar{C}_{\text{bank}}$ [S8/case_28] (canonical 33.90 cyc with SHL overhead; 27.79 cyc lower bound for pure ld.shared at [S8/case_52] — see Ch2 §2.1 dual-measurement table). The register term accounts for ldmatrix feeding cost, which is pipelined behind MMA at ILP $\geq$ 2 [S9/case_10].

13.1.3 Synchronization Time

$$T_{\text{sync}} = \sum_{i} N_{\text{barrier},i} \times C_{\text{barrier}}(\text{type}_i,\; \text{scope}_i,\; W) \;+\; \sum_{j} N_{\text{fence},j} \times C_{\text{fence}}(\text{scope}_j)$$

13.1.4 Launch and Tail Overhead

$$T_{\text{launch}} = \begin{cases} 5.4\;\mu s & \text{warm host} \\ 1.5\;\mu s & \text{GPU-side} \\ 0.12\;\mu s & \text{CUDA graph} \end{cases}$$
$$T_{\text{tail}} = \frac{N_{\text{tiles}} \bmod N_{\text{SM}}}{N_{\text{SM}}} \times T_{\text{per\_tile}}, \quad N_{\text{SM}} = 110$$

The tail effect arises from wave quantization on 110 SMs. When the grid size is not an exact multiple of $N_{\text{SM}}$, the final wave under-utilizes the GPU [S14/case_29].

13.2 Multi-Resource Bottleneck Framework

Beyond the simple max-of-three model, production kernels exhibit simultaneous contention across multiple pipes. The multi-resource formulation accounts for six independent hardware resources:

$$T_{\text{kernel}} = \max(T_{\text{tc}},\; T_{\text{alu}},\; T_{\text{sfu}},\; T_{\text{lsu}},\; T_{\text{smem}},\; T_{\text{sync}}) \times (1 + \epsilon_{\text{overhead}})$$

For N-way instruction mixes, the model applies:

$$T_{\text{mix}} = \max(T_{\text{pipe}_A},\; T_{\text{pipe}_B},\; \ldots) \times \left(1 + \sum_{k} \delta_{\text{switch},k}\right)$$

The switching penalty $\delta_{\text{switch}}$ is derived from the co-issue matrix (Table 4 in Section 13.4).

TMA as a seventh resource bottleneck: TMA throughput peaks at 14.50 B/cyc ([S26/case_35]), representing a seventh potential bottleneck not captured by the six-pipe model above. At ILP=4, TC consumption is 10x the TMA delivery rate ([S26/case_36]). For TC-bound kernels with TMA pipelining, the practical bottleneck is often TMA delivery, not TC execution.

If $\text{AI} < \text{AI}_{\text{crossover}}$, the kernel is memory-bound; if $\text{AI} > \text{AI}_{\text{crossover}}$, the kernel is compute-bound.

13.4 Occupancy and Scheduling Model

The warp scheduler's throughput scaling follows a measured saturation curve:

$$\text{throughput}_{\text{eff}}(W,\; \text{ILP}) = \text{throughput}_{\text{base}} \times \min\!\left(1.0,\; \frac{W \times \min(\text{ILP},\; 4)}{16}\right)$$

Key parameters:

Parameter Value Source
Max warps per SM 48 S3/case_2
Max blocks per SM 24 S3/case_2
Register file per SM 65536 x 32-bit S3/case_2
TC saturation threshold 16 warps or ILP >= 4 S11/case_61
ALU saturation threshold 16 warps @ ILP >= 4 S21/case_53
Issue width 1 warp-instruction per cycle per sub-partition S11/case_3
Sub-partitions per SM 4 S3/case_2

Occupancy is determined by the most constraining resource:

$$\text{occupancy} = \frac{1}{48} \min\!\begin{cases} \left\lfloor \dfrac{65536}{R_{\text{thread}} \times 32} \right\rfloor \times 32 & \text{(register-limited)} \\[6pt] \left\lfloor \dfrac{S_{\text{avail}}}{S_{\text{block}}} \right\rfloor \times W_{\text{block}} & \text{(smem-limited)} \\[6pt] 48 & \text{(absolute max warps)} \\[6pt] 24 \times W_{\text{block}} & \text{(block-limited)} \end{cases}$$

13.5 Prediction Workflow

Given a kernel specification, the model predicts execution time through the following steps:

Step 1: Extract Instruction Mix

From the kernel source or NCU profile, extract:

  • MMA instruction count and precision
  • ALU instruction count (fmalite vs fmaheavy)
  • SFU instruction count
  • Load/store instruction counts and access patterns
  • Synchronization primitives and their types

Step 2: Compute Per-Resource Demand

Apply the formulas from Section 13.2 to compute T_tc, T_alu, T_sfu, T_lsu, T_smem, T_sync independently.

Step 3: Apply Co-Issue Overlap

For each pair of co-executing pipes, apply the overlap ratio from Table 4 to determine the effective parallel execution time:

$$T_{\text{eff}} = \max(T_A,\; T_B) + (1 - r_{\text{overlap}}) \times \min(T_A,\; T_B)$$

Step 4: Add Fixed Overheads

$$T_{\text{total}} = T_{\text{eff}} + T_{\text{launch}} + T_{\text{tail}}$$

13.8.1 GPU Performance Equation (CAQR §1.10 Adapted)

The GPU-adapted performance equation, validated against Section 25 measurements:

$$T_{\text{kernel}} = \frac{N_{\text{insn}} \times \overline{\text{CPI}}}{N_{\text{SMSP}}} = \frac{\sum_i N_{\text{pipe}_i} \times \text{CPI}_{\text{pipe}_i}}{N_{\text{SM}} \times 4}$$

where $N_{\text{pipe}_i}$ is the instruction count for pipe $i$ (from NCU smsp__inst_executed_pipe_*), $\text{CPI}_{\text{pipe}_i}$ is cycles per instruction from microbenchmarks, and $N_{\text{SMSP}} = 110 \times 4 = 440$ issue slots.

$$T_{\text{predicted}} = \frac{460\text{M} \times 9.53}{440} = 9.95\text{M cyc} \;\rightarrow\; 3.98\text{ ms @ 2.5 GHz}$$

13.8.3 Little's Law for Memory Subsystem

MLP = Throughput × Latency = Queue_depth

Measured (L2 path, S25/case_4):
  N=16 independent loads → MLP = 13.7
  L2 latency = 389 cyc
  Throughput = 16/455 = 0.0352 loads/cyc
  Little's Law: 0.0352 × 389 = 13.7 ✓ (matches measured queue depth 12-13)

13.8.4 AMAT for Memory Hierarchy

AMAT = 44 + MR_L1 × (310 + MR_L2 × 522)

L1 hit = 44 cyc (constant, pattern/stride independent; S26/case_7-9)
L2 penalty = 354 - 44 = 310 cyc
DRAM penalty = 522 cyc (unchanged)

Validated against NCU hit rates for GEMM (S25/case_34):
  L1 hit rate = 73.86% → MR_L1 = 0.2614
  L2 hit rate = 89.71% → MR_L2 = 0.1029
  AMAT = 44 + 0.2614 × (310 + 0.1029 × 522) = 44 + 0.2614 × 363.7 = 139.1 cyc

Note on DRAM latency: The 876 cyc DRAM value used in prior measurements could not be reproduced ([S26/case_11]). GDDR7 page-miss penalty is only ~10 cyc (362 vs 372 cyc); GDDR7 channel interleaving neutralizes row-buffer miss penalties. The practical DRAM latency is 362+/-10 cyc.
