// Expected: 1.92x over serial [S10/case_82]
// Stage 0: TMA prefetch (479 cyc latency hidden by stages 1-2)
// Stage 1: Compute on buffer[i-1]
// Stage 2: Wait for buffer[i], compute on buffer[i]
for (int stage = 0; stage < K_TILES; stage++) {

Pattern P3: Fused Normalization + Activation (Memory-Bound) 

// Expected: 1.28-1.41x over separate kernels [S19/case_44]
// Eliminates 1 DRAM round-trip (~1231 GB/s BW saved)
__global__ void fused_rmsnorm_silu(half* out, const half* in,
                                    const half* weight, int H) {
    // Phase 1: warp-level reduction for variance [S7/case_70: 170 cyc FP32]
    float sum_sq = warp_reduce_sum(x * x);  // redux.sync: 45 cyc [S7/case_8]
    float rms = rsqrtf(sum_sq / H + eps);    // MUFU.RSQ: 40 cyc [S5/case_1]
    // Phase 2: normalize + SiLU in single pass
    out[i] = (half)(x * rms * weight[i] * sigmoid(x * rms * weight[i]));
    // SiLU memory-bound at production sizes: ~1121 GB/s [S7/case_78]
}

Phase 3: Regression Check

Compare against baseline (cuBLAS or previous version):

speedup = T_baseline / T_optimized

IF speedup < 1.0:
    -> Regression detected; revert and re-diagnose

IF speedup > expected * 1.5:
    -> Suspiciously high; check for measurement artifact (Chapter 13 methodology)

IF speedup within [expected * 0.8, expected * 1.2]:
    -> Optimization successful; commit

Feedback Loop

IF validation fails:
  1. Re-profile with NCU (Chapter 13 methodology)
  2. Re-classify bottleneck (Section 15.3 -- bottleneck may have shifted)
  3. Select next-priority strategy from Section 15.4 (avoid already-tried strategies)
  4. Regenerate kernel
  5. Re-validate
  Maximum iterations: 3 (to avoid optimization thrashing)

15.11 Optimization Priority Rules

The agent must apply optimizations in priority order. The following rules determine when to stop:

  1. P0 strategies first: Always apply all applicable P0 strategies before considering P1.
  2. Never violate hard constraints: Even if a P0 strategy would improve performance, skip it if it would violate C1-C14.
  3. Check soft constraints after each change: If a P0 strategy triggers a soft constraint violation (e.g., I-cache cliff from excessive unrolling), back off to the threshold.
  4. Stop at 3 iterations maximum: If the validation loop has run 3 times without convergence, escalate to human review.
  5. Stop at <10% headroom: If predicted headroom is below 10%, the kernel is near-optimal and further optimization is unlikely to yield significant gains.
  6. Regime-specific ordering: The priority order depends on the diagnosed bottleneck regime:
  7. TC-bound GEMM: S1/S2 (ILP) -> S3 (swizzle) -> S14 (pipeline) -> S19/S20 (precision)
  8. DRAM-bound GEMV: S7 (coalescing) -> S4 (vectorize) -> S5 (quantize) -> S18 (fuse)
  9. Mixed attention: S9 (MMA rowsum) -> S21 (selp mask) -> S3 (swizzle) -> S14 (pipeline)
  10. Launch-limited: S15 (persistent) -> S16 (graphs) -> S18 (fuse)
  11. Sync-limited: S10 (named barrier) -> S11 (CTA fence) -> S12 (relaxed arrive)
