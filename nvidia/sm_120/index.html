Key Architectural Findings (Top 10)

  1. 2-Bank RF with Zero Observable Penalty (Ch.3): SM_120 has a 2-bank (R%2 parity) register file with +1 cyc conflict penalty — but ptxas's parity-aware allocation ensures conflicts never appear in production code.

  2. Dual-Subpipe FMA Architecture (Ch.3,5): IADD3 (fmalite) and FFMA (fmaheavy) execute on fully independent subpipes at MIX/MAX=1.001, enabling 2 ops/cycle aggregate for mixed INT+FP.

  3. TC Pipeline Depth 1.81 / 4.09 (Ch.4): HMMA/QMMA share a 2-slot pipeline (depth 1.81), while IMMA has a separate 4-slot pipeline (depth 4.09) with 2.4× higher throughput.

  4. TMA Multicast Absent (Ch.6): GB202 lacks the LRC multicast crossbar. All multicast attempts are SW-emulated at 2400-7500× penalty. CUTLASS/FlashInfer enforce ClusterShape<1,1,1>.

  5. Memory Wall at L1→L2 (Ch.2): The 8.0× latency cliff from L1 (44 cyc) to L2 (354 cyc) is the true memory wall. L2→DRAM is nearly transparent (1.01× for random access).

  6. Stall Counter Trust Model (Ch.7,12): stall < write-back latency causes silent data corruption. No trap, no CUDA error. Only stall=0 triggers the hardware safety net.

  7. Pipelined Bank Conflicts (Ch.2): 32-way smem bank conflict costs only 1.43× throughput (vs 32× on prior architectures), enabled by a pipelined replay arbiter.

  8. DSMEM Is Chip-Wide (Ch.6): Cluster CTAs are placed cross-GPC (stride-20). DSMEM latency is uniform regardless of SM distance — the SM2SM fabric is a flat interconnect.

  9. Softmax Is Architecturally Free (Ch.8): MUFU.EX2 co-issues 100% with TC operations. In FlashAttention, the 25% wall-time softmax cost is fully overlapped, adding ~0% actual latency.

  10. ILP Dominates Over Occupancy (Ch.7): Single warp at ILP=4 achieves 92% of peak FP32 throughput. Additional warps help only for memory latency hiding, not compute.
